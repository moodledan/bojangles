★ Setup

Grunt and grunt plugins are installed and managed via npm, the Node.js package manager.

Get those beauties installed, then from within the bojangles directory run: 

npm install -g grunt-cli
npm install
grunt

★ Winning

The src directory is the raw (fully functional) source for the project, you should be able to open index.html inside this dirctory in your browser - Successfully if your reading this : )

After running Grunt you'll find a new build directory, which is the processed output of src as well as a node_modules directory containing lots of Grunt goodness.

Now what ...

You run grunt and the Gruntfile.js automates a raft of, what would usually be, manual processes. The processes built into the Gruntfile.js are outlined below in the order they occur:

★ Clean

Everytime we run grunt we destroy the build directory and re-create it. Clean is responsible for this.

★ Targethtml

Index.html in the src directory contains some HTML which is relevant to building your app, such as links to un-minfied CSS and JS as well as commented links to concatenated and minified CSS and JS - Which you would use in your build. Using targethtml we can switch which code is referenced and for which type of build. targethtml also adds version no. query string to our CSS and JS to prevent caching when an update is released.

★ Copy

The name says it all, copies src files into the build directory. Once specific example is copying JS libraries such as jQuery and Modenizer, which comes with minified source, to the build directory.

★ Imageoptim

Image optimisation, in this case we first let copy do its job and then optimise the images in the build directory. Personal preference, leaving the original images un-touched. You'll need to have ImageOptim installed for this to work correctly.

★ Concat

Grab all our JS and concatenate the source into app.js, remember targethtml handles the switching of filenames and paths in index.html to reflect the new concatenated JS file app.js

★ Uglify

Minification of the concatenated JS as well as making it 'ugly'. Note this will probably need updating as more JS is added, see the file declaration for cssmin to see how to specify files in order if the load order of your JS is important.

★ Cssmin

cssmin minifies the CSS as well as combining it into a single file. Note that we specify each seperate CSS file in its correct order as is necessary with CSS. As with uglify targethtml handles the file name and path changes for CSS as well as JS.

★ Jshint

Lint our JS, only JS in the top level of the src/js directory as were concered with the JS we've written. We lint the un-minified JS from the src directory rather than the minified source. The libs directory contains jQuery and Modernizer lets assume they pass, if they don't what hope do the rest of us have...

★ Csslint

Lint our CSS, we lint the un-minified css from the src directory rather than the minified source.

★ Build actions

We have a default action defined, which can be run like so: grunt from the Terminal. This runs the following, clean, targethtml, copy, concat, uglify, cssmin.

We have an action to lint CSS and JS like so: grunt lint this lints the src not the build output.

Lastly to optimise images we have grunt compressimages which will copy the img directory from src directory to the build directory and the compress images.
